(inner) loadVideo(streamInformationObject, callback)

Streams a video file to all other clients/peers.
Parameters:
Name Type Description
streamInformationObject StreamInformationObject This object contains all data that is needed to initiate loading the video from other peers and/or a Web server. StreamInformationObjects can be created by the streamVideo method.
callback OakStreaming~loadedVideoFinished This callback gets called when the video has been loaded entirely into the buffer of the video player.

(inner) streamVideo(videoFile, optionsopt, callback)

Streams a video file to all other clients/peers.
Parameters:
Name Type Attributes Description
videoFile object The video file that should be streamed to the other clients/peers. This paramter can either be a W3C File object, a W3C FileList, a Node Buffer object or a Readable stream object.
options object <optional>
 Options for the creation of the StreamInformationObject, that gets passed as an argument to the callback function.
Properties
Name Type Description
XHRPath string The path that will be used for the XML HTTP Request (XHR). If the option object or this property of the option object is undefined, no video data will be requested from the server.
callback OakStreaming~streamVideoFinished This callback function gets called with the generated StreamInformationObject at the end of the execution of streamVideo.

StreamInformationObject

Type:

Properties:
Name Type Description
magnetURI string Magnet URI of the torrent. If this property is undefined, no video data will be requested from the WebTorrent network.
videoFileSize number The size in byte of the video file that was passed as an argument.
XHRPath string The file path (e.g. /example.mp4 or /videos/example2.mp4) that will be used for the XML HTTP Requests to the Web server. Via these XML HTTP Requests, video data will be requested from the Web sever. If this property is undefined, no video data will be requested from the Web server.
